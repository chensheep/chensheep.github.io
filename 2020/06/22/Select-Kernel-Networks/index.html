Select Kernel Networks

Posted on 2020-06-22 Edited on 2020-06-23 Views:

論文請參考 arXiv:1903.06586 。

介紹

Select Kernel Networks（SKNets）於2019提出，為ResNet後在CNN上的改進。

主要提出解決過去CNN網路Receptive Field都是固定大小，但實際上的資料集，一張圖中目標物件的大小不一定是一樣的，所以這篇論文提出一個可以動態選擇使用哪種Receptive Field有較好預測結果的方法。

Selective Kernel (SK) unit在這篇論文中被提出，SK unit使用多個分支，分別為不同的卷積核大小，最後一不同的權重將所有分支合在一起，非常類似SENet的做法，只是SENet是對不同的通道去加權。

最後將Selective Kernel (SK) unit堆疊產生Selective Kernel Network (SKNets)。

另外作者也做了實驗，將樣本圖片中的目標物件放大，可觀察到相對大的卷積核可獲取更有效的資訊。

接著我們解釋幾個論文個重點

Selective Kernel (SK) unit

參考Fig.1。，SK unit主要由三部份組成，Split、Fuse與Select，在這張圖中Split部分為兩個分支，實際上可分成更多的分支，使用不同的卷積核。

Split

在這裡使用兩個分支，其一為3x3卷積，另一個為5X5卷積，裡面都包含Batch Normalization與ReLU。
另外關於5X5卷積部分，不是使用一般的5X5卷積，為減少運算量，使用了空洞卷積(Dilated Convolution)，大小為3，dilation大小為2。
完成卷積後將兩個特徵圖element-wise相加，也保留這些特徵圖到Select使用。

Fuse

這部分跟SENet做法類似，主要分成兩個步驟

  1. Global Average Pooling
  2. Fully Connected Layer
    第一步針對Ｕ每個channel做Global Average Pooling產程Ｓ，Ｓ維度為（C X 1)
    第二步 W X S 其中Ｗ維度為（d X C）再經過ReLU產生Ｚ（d X 1），作者指出目的為減少運算量，我認為也有壓縮凸顯特徵的功用類似word embeding的概念。
    d是如何決定的，這邊作者設定為max(C/r, L)最小為Ｌ=32，設定r去做實驗。

Select

獲得Ｚ之後，目的要算出對於在Split階段使用不同RF產生出來的特徵圖的權重。
在此我們有兩個分支，所以需產生出兩個向量分別為a、b，與Fuse中產生Z的方法類似，Ｗa X Z 與 Ｗb X Z分別產生兩個向量後，因為我們目的在於產生兩個特徵圖的權重，所以在針對每個通道去做softmax即可以獲得在每個通道上兩張特徵圖所佔的比例。
最後得到a、b向量後再分別與兩個分支的特徵圖做element-wise相乘，再將乘玩的兩個特徵圖element-wise相加。
