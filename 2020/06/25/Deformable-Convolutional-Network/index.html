Deformable Convolutional Network

Posted on 2020-06-25 Edited on 2020-06-30 Views: Disqus:

論文請參考 arXiv:1703.06211。

介紹

論文中一開始提到，由於CNNs固定的幾何結構，限制了模型對於幾何轉換的能力。

論文提出兩個具體的方解決這個問題，分別為deformable convolution與deformable RoI pooling，兩個方法都是藉由增加額外的可學習的offsets，去調整空間取樣的位置，並且都能輕易地移植到當今流行的CNNs中，使用back-propagation去訓練offsets。

官方釋出的程式碼可參考連結。

Deformable convolution

{Fig.2}

Fig.2 中，最左邊(a)為傳統的卷積方法，(b)、(c)與(d)分別是各種Deformable convolution，從圖中可見傳統的卷積(a)中取樣點為綠色點，(b)、(c)與(d)則是加上淺藍色的各種方向的offsets，得到新的深藍色取樣點，至於offsets要如何產生，見Fig.3。

{Fig.3}

Fig.3中增加了offset的分支，用來產生offset，這邊也可以觀察到分支中有卷積的操作，代表offset是藉由輸入的特徵圖學習產生出來。

Fig.4

符號的部分，Ｒ定義了感受野（receptive field）的大小與空洞值（dilation），卷積核大小為3X3空洞值為1的卷積可表示為Fig.4，Fig.5呈現得更清楚。

Fig.5

卷積操作可表示為Fig.6，其中y(p0)表示在輸出特徵圖y的p0這個位置，w(pn)表示pn的權重，x(p0+pn)表示x中p0+pn的位置。

Fig.6

Deformable convolution中，需針對原本的取樣點去加上offeset的動作，因此表示為Fig.7，∆pn為每個點的offeset。

得到offeset之後，將原本的取樣點座標加上offset，做Deformable convolution輸出特徵圖。至於取樣座標不一定為整數，不為整數時則是使用雙線性差值(bilinear interpolation)去計算像素值。
